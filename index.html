Pre-LLM Next-Word Prediction

Old n-gram models guess the next word from raw frequency alone. Write a passage below with one or more blanks marked <blank>, and watch a thousand guesses come out of the machine for each hole — then see how a real language model handles the same holes.

Set up the passage

Embeddings and RNN below train their background signal on a bundled real-text sample (~1GB of Wikipedia text, split into chunk files in static-corpus/ and shipped with this site — no external API call). The n-gram models (unigram/bigram/trigram/4-gram) always query a real, live, pretrained n-gram corpus — infini-gram, a free public API that holds an index over Dolma-v1.7 (~2.6 trillion tokens) and returns exact next-word statistics in milliseconds. Your story is still trained locally so a well-attested story-specific pattern can win out; infini-gram just supplies what the model backs off to.

Pick models to compare

Six stages of the same field, each on the same blank: unigram → bigram → trigram/4-gram (frequency tables, 1980s–2000s, now blended live with real statistics from a trillion-token corpus via infini-gram) → embeddings (words as vectors from co-occurrence statistics, the word2vec/GloVe-era idea, 2003–2013) → RNN (a running hidden state instead of a fixed window, Mikolov 2010) → LLM (a real pretrained transformer using full bidirectional context, 2018–present).

The LLM option downloads a ~65 MB model the first time (cached after that) and runs it fully in your browser — no server, no API key.

The unigram / bigram / trigram / 4-gram models above always query a real pretrained n-gram corpus live, via infini-gram (Liu, Min, Zettlemoyer, Choi & Hajishirzi, 2024) — a free public API backed by a suffix-array index over Dolma-v1.7 (~2.6 trillion tokens of web text, books, Wikipedia, Reddit, and code), returning exact next-word counts and probabilities in milliseconds — no download, no key. That's what actually replaced the old small local/sampled background corpus for these four models.

The embeddings and RNN models train their background signal on a real text sample bundled directly with this site — Wikipedia text, split into small chunk files under static-corpus/ so no single file approaches GitHub's size limits, and no third-party API call is needed at runtime.
