Pre-LLM Next-Word Prediction

Old n-gram models guess the next word from raw frequency alone. Write a passage below with one or more blanks marked <blank>, and watch a thousand guesses come out of the machine for each hole — then see how a real language model handles the same holes.

Set up the passage

Every model below — n-grams (unigram/bigram/trigram/4-gram), embeddings, and RNN — trains its background signal on a bundled real-text sample (~1GB of Wikipedia text, split into chunk files in static-corpus/ and shipped with this site — no external API call, no live network dependency). Your story is trained alongside it and repeated so a well-attested story-specific pattern can still win out over the generic background.

Pick models to compare

Six stages of the same field, each on the same blank: unigram → bigram → trigram/4-gram (frequency tables, 1980s–2000s, trained here on your story blended with a bundled real-text background corpus) → embeddings (words as vectors from co-occurrence statistics, the word2vec/GloVe-era idea, 2003–2013) → RNN (a running hidden state instead of a fixed window, Mikolov 2010) → LLM (a real pretrained transformer using full bidirectional context, 2018–present).

The LLM option downloads a ~65 MB model the first time (cached after that) and runs it fully in your browser — no server, no API key.

All six models — unigram, bigram, trigram, 4-gram, embeddings, and RNN — train their background signal the same way: on a real text sample bundled directly with this site — Wikipedia text, split into small chunk files under static-corpus/ so no single file approaches GitHub's size limits, and no third-party API call is needed at runtime. Your own story is trained alongside it (repeated a few times so it isn't statistically swamped), so a well-attested story-specific pattern can still win out over the generic background.
